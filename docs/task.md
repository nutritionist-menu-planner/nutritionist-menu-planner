## Phase 1. 데이터베이스 설계 및 구축

### Epic 1-1. 도메인 모델 정의 및 문서화

**목표**: 비즈니스 도메인을 데이터 구조로 변환

- [ ] **Task 1-1-1**: 식단(Meal Plan) 엔티티 정의
  - 월별 식단, 일일 식단 구조 결정
  - 식단 메타데이터 (작성일, 작성자, 대상 인원 등)
  - 식단 상태 (초안/확정/배포) 정의

- [ ] **Task 1-1-2**: 식단 구성 요소(Meal Item) 정의
  - 밥/국/반찬/후식 카테고리 정의
  - 1인분 기준 분량 단위 설정
  - 조리 방법 정보 포함 여부 결정

- [ ] **Task 1-1-3**: 식재료(Ingredient) 엔티티 정의
  - 식재료 기본 정보 (이름, 단위, 카테고리)
  - 대체 식재료 관계 설정
  - 계절성/수급 안정성 속성

- [ ] **Task 1-1-4**: 영양 정보(Nutrition Info) 구조 정의
  - 필수 영양소 목록 (열량, 탄수화물, 단백질, 지방, 나트륨 등)
  - 100g 기준 vs 1인분 기준 선택
  - 영양소 출처 데이터 (식품의약품안전처 DB 연동 계획)

- [ ] **Task 1-1-5**: 알레르기(Allergen) 및 제한 식품 모델 정의
  - 식약처 표시 대상 알레르기 유발 식품 목록
  - 종교적/문화적 제한 식품 (할랄, 코셔 등) 포함 여부
  - 식재료-알레르기 매핑 구조

**목표**: 관계형 데이터베이스 구조 완성

- [ ] **Task 1-2-3**: 식단 이력 관리 구조 설계

  - 수정 사유 기록 구조
  - 롤백 가능 여부 결정

- [ ] **Task 1-2-4**: ERD 다이어그램 작성 및 리뷰
