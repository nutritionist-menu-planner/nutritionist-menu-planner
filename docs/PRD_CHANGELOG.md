# PRD 변경 이력 (PRD Changelog)

## 목적
PRD.md의 모든 변경 사항을 추적하여 프로젝트 범위 및 요구사항 변화를 문서화합니다.

---

## 변경 이력

### [v1.0.0] - 2026-01-06

**상태**: ✅ 최종 승인됨

**작성자**: 프로젝트 팀

**주요 내용**:
- 영양사 대상 식단 자동 생성·관리 플랫폼 PRD 초안 작성
- MVP 범위 확정: 영양사 전용 기능으로 한정
- 제외 기능 명시: 개인 맞춤 다이어트, 벌크업, B2C 서비스

**핵심 기능 요구사항**:
1. 기존 식단 분석 기반 자동 생성
2. 급식일 자동 계산 및 휴무일 반영
3. 영양사 주도 실시간 식단 수정
4. 식재료 파일 업로드 및 관리

**비기능 요구사항**:
- 영양사 책임 구조 존중
- 계산 결과의 설명 가능성
- 엑셀 대비 낮은 학습 곡선
- 데이터 이력 관리 및 보안

**성공 지표 (KPI)**:
- 식단 편성 소요 시간 감소율
- 영양사 수정 빈도 감소
- 월간 활성 영양사 수
- 재사용률

**이해관계자 승인**: ✅ 완료

**검토 의견**:
- PRD 내용이 명확하고 실행 가능한 수준으로 작성됨
- MVP 범위가 적절히 설정됨
- Out of Scope 기능이 명확히 정의됨

## 향후 변경 프로세스

### 변경 요청 절차
1. 변경 필요성 및 배경 문서화
2. 팀 리뷰 및 논의
3. 영향도 분석 (개발 일정, 리소스)
4. 승인 후 PRD.md 업데이트
5. 본 문서에 변경 이력 기록

### 변경 이력 템플릿
```markdown
### [vX.X.X] - YYYY-MM-DD

**상태**: 검토 중 / 승인됨 / 반려됨

**작성자**: [이름]

**변경 사유**:
[변경이 필요한 이유]

**변경 내용**:
- [변경 항목 1]
- [변경 항목 2]

**영향도**:
- 개발 일정: [영향도 설명]
- 기술 스택: [영향도 설명]
- 리소스: [영향도 설명]

**승인 여부**: ✅ / ❌

**검토 의견**:
[검토자 의견]
```
